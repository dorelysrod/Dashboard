# Accessibility (WCAG 2.2 AA)

Accessibility is part of the design bar, not a separate pass. Enforced by `ux-accessibility` (design-time) and `testing-accessibility` (automated, CI-gated).

## Baseline requirements

- Color contrast: at least 4.5:1 for normal text, 3:1 for large text/UI components (WCAG AA).
- All interactive elements are keyboard-reachable and operable (Tab/Shift+Tab/Enter/Space/Esc as appropriate).
- Focus is always visible (no `outline: none` without a replacement focus style).
- All images convey meaning via alt text, or are marked decorative (`alt=""`).
- Forms: every input has an associated, visible label (not placeholder-only).
- Headings form a logical hierarchy (no skipped levels) for screen-reader navigation.
- Motion: respect `prefers-reduced-motion` for non-essential animation.

## Process

1. `ux-accessibility` reviews new screens/components at design time against this list.
2. `testing-accessibility` runs automated checks (axe-core or equivalent) in CI — zero "serious"/"critical" violations on `demo-critical` pages blocks merge.
3. Manual spot-check before `/demo-prep`: navigate the primary demo journey using only the keyboard.
