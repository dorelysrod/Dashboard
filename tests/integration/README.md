# /tests/integration

API + data-layer integration tests, and tests for any cross-feature interaction through `/shared`. Generated by `integration-test-generation`.

## Coverage structure

- **Happy path** — success response for each endpoint
- **Edge cases** — boundary inputs to the endpoint (see `edge-case-generator`)
- **Failure cases** — at least one documented error response per endpoint (4xx/5xx)
- **Regression** — added when `/new-bug` identifies an integration-level bug

Empty until the stack and first API endpoints are chosen at Hour 0.
