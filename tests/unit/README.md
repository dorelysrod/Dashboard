# /tests/unit

Cross-cutting unit test utilities and fixtures, plus unit tests for `/shared` code. Most unit tests live alongside the code they test inside `/features/<name>/tests/`. Generated by `unit-test-generation`.

## Coverage structure

- **Happy path** — at least one test for the expected/common case
- **Edge cases** — boundary values, empty/null inputs (see `edge-case-generator`)
- **Failure cases** — invalid input, thrown errors, rejected promises
- **Regression** — when `/new-bug` fixes a bug, its regression test is added here or in the feature's test folder, named to reference the issue (e.g. `bug-123.test.ts`)
