# /tests/e2e

End-to-end user-journey tests, generated by `e2e-test-generation`. `demo-critical` journeys (from `ux-user-journey` and the demo script) are prioritized — see `/demo-prep`.

## Coverage structure

- **Happy path** — the primary journey from entry point to outcome
- **Edge cases** — alternate paths through the journey (see `edge-case-generator`)
- **Failure cases** — what the user sees when a step fails (network error, validation error)
- **Regression** — added when `/new-bug` identifies a journey-level bug

## Conventions

- Use accessible selectors (role/label/text), not brittle CSS selectors — see `ux-accessibility`.
- Name files after the journey, e.g. `upload-and-itemize.spec.ts`.

Empty until the stack is chosen at Hour 0.
